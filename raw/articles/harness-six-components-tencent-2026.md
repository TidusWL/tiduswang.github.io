# 一文讲透如何构建 Harness——六大组件全解析

> 来源：腾讯云开发者社区 | 收录日期：2026-06-03

## 核心公式

**Agent = Model + Harness**
模型提供智能，Harness 让智能变得有用。

## 裸模型四大硬伤

| 硬伤 | 缺失的能力 | Harness 对应组件 |
|---|---|---|
| 无法维持跨会话状态 | 长期记忆 | 文件系统 + AGENTS.md |
| 无法执行代码 | 行动能力 | Bash + 沙箱 |
| 无法获取实时知识 | 感知能力 | Web搜索 + MCP |
| 无法搭建工作环境 | 环境操控 | 文件系统 + 上下文工程 + 编排 |

## 六大组件

### 1. 文件系统

三大核心能力：工作空间与中间结果存储、Agent协作基础（共享白板）、版本追踪与错误回滚。

设计原则：目录结构清晰、文件粒度适中、元数据丰富。

### 2. Bash + 沙箱

自我验证循环：写 → 跑 → 看 → 修 → 再来。

测试数据：具备自我验证循环的 Agent 任务完成率比"一次性生成"高出 40%-60%。

沙箱安全机制：资源限制、网络隔离、文件系统隔离、超时自动终止。

沙箱技术选型：Docker容器、gVisor/Firecracker、WebAssembly (WASM)、Nix。

### 3. 记忆（AGENTS.md）

核心洞察：上下文注入 = 不改权重给模型加知识。

工作机制：写入 → 存储 → 注入。

最佳实践：层次化存放、结构化书写、定期清理、双向可编辑。

### 4. Web搜索 + MCP

MCP（Model Context Protocol）= AI 世界的 USB 接口。

Web搜索关键问题：查询构建、结果筛选、内容提取、信息整合。

### 5. 上下文工程

Context Rot（上下文腐烂）四大表现：信噪比下降、矛盾信息累积、Token 浪费、推理质量退化。

核心策略：压缩、工具输出卸载、Skills 渐进加载、分层上下文结构。

### 6. 编排 + Hooks

编排解决：子Agent调度、任务分发、模型路由、结果聚合。

编排模式演进：线性管道 → DAG编排 → 动态编排 → 层级编排。

Hooks 典型应用：Lint 检查、续接、格式约束、安全过滤、成本控制。

核心价值：用确定性规则约束概率性输出——"概率性生成 + 确定性校验"。

## System Prompt

四大角色：定义角色边界、注入领域知识、约束安全规则、贯穿所有组件。

System Prompt 装"必须知道的"，AGENTS.md 装"最好知道的"。

## 关键结论

模型决定了 Agent 能力的下限，Harness 决定了 Agent 能力的上限。
