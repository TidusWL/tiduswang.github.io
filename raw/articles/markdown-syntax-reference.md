# Markdown 语法完整参考

> 来源：roc-mountain.github.io/Markdown | 收录日期：2026-06-03

## 概述

Markdown 是一种轻量级标记语言，由 John Gruber 于 2004 年创建。使用易读易写的纯文本格式编写文档，可与 HTML 混编，可导出 HTML、PDF 以及 .md 格式。

**特点**：纯文本可移植、独立于平台、兼容 HTML、语法简洁（常用标记不超过10个）。

**文件后缀**：`.md`

## 标题

### 方式一：Atx 形式（推荐）

在行首插入 1-6 个 `#`，对应 H1-H6：

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

> 可选择在行尾加上 `#`，数量不必与开头一致。

### 方式二：Setext 形式（仅支持两级）

```markdown
一级标题
========

二级标题
--------
```

## 段落与换行

- **段落**：一个或多个连续的文本行，前后需有一个以上空行
- **换行**：在行末输入两个以上空格，然后回车

```markdown
这是第一行（行末有两个空格）  
这是第二行

这是新段落（中间有空行）
```

> 普通段落不该用空格或制表符缩进（缩进4个空格以上会变成代码区块）。

## 强调

| 效果 | 语法 | 示例 |
|---|---|---|
| *倾斜* | `*文本*` 或 `_文本_` | `*倾斜*` |
| **加粗** | `**文本**` 或 `__文本__` | `**加粗**` |
| ***倾斜加粗*** | `***文本***` | `***倾斜加粗***` |
| ~~删除线~~ | `~~文本~~` | `~~删除线~~` |

> 强调可以嵌套，如 `**这是 *加粗倾斜* 文本**`。

## 列表

### 无序列表

使用 `*`、`+` 或 `-` 作为列表标记（后面加一个空格）：

```markdown
* 项目一
* 项目二
  * 子项目（缩进2-3个空格）
  * 子项目
* 项目三

+ 也可以用加号
- 或用减号
```

### 有序列表

数字接着一个英文句点（任意数字开头均可）：

```markdown
1. 第一项
2. 第二项
3. 第三项
   1. 子项目
   2. 子项目
7. 数字不必从1开始（渲染时会自动编号）
```

### 任务列表（GFM 扩展）

```markdown
- [x] 已完成任务
- [ ] 未完成任务
- [ ] 另一个任务
```

## 引用

使用 `>` 标记，每行或每段开头均可：

```markdown
> 这是一段引用
> 
> 这是引用的第二段
>
> > 嵌套引用（二级引用）
> > > 可以继续嵌套
```

引用内可使用其他 Markdown 语法：

```markdown
> ## 引用内的标题
> 
> - 引用内的列表
> - **引用内的加粗**
```

## 代码

### 行内代码

用单个反引号包裹：

```markdown
使用 `git commit` 提交代码
```

### 代码块

用三个反引号包裹，第一行可指定语言实现语法高亮：

````markdown
```python
def hello():
    print("Hello, World!")
```
````

效果：

```python
def hello():
    print("Hello, World!")
```

### 缩进式代码块

每行缩进4个空格或1个制表符：

```markdown
    function test() {
        console.log("Hello");
    }
```

## 链接

### 行内式

```markdown
[链接文字][标记]
```
示例：
```markdown
[Markdown 官网](https://markdown.cn)
[GitHub](https://github.com "代码托管平台")
```

### 参考式

```markdown
[链接文字][标记]

[标记]: https://github.com "鼠标悬停提示"
```

> 参考式标记可以放在文档任意地方（通常在文末统一管理）。

### 自动链接

```markdown
<https://example.com>
<email@example.com>
```

## 图片

语法与链接相似，前面加 `!`：

```markdown
![替代文本](图片URL "可选标题")
![本地图片](./images/logo.png)
```

> 建议将图片放在项目目录下统一管理，使用相对路径引用。

## 表格

用 `|` 分隔列，`-` 分隔表头与内容，`:` 控制对齐：

```markdown
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| 内容1  | 内容2 | 内容3  |
| 内容4  | 内容5 | 内容6  |
```

效果：

| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| 内容1  | 内容2 | 内容3  |

### 快捷写法

可以直接写表头行，用 `-` 分隔，无需手动对齐：

```markdown
表头1 | 表头2 | 表头3
------|------|------
内容1 | 内容2 | 内容3
```

可以在单元格内使用其他 Markdown 语法。

## 分隔线

行内三个以上的 `*`、`-` 或 `_`：

```markdown
---
***
___
- - -
* * *
```

效果：

---

## HTML 支持

Markdown 兼容 HTML，可以直接使用 HTML 标签：

```markdown
<kbd>Ctrl</kbd> + <kbd>C</kbd>
<details>
<summary>点击展开</summary>
这里是折叠内容
</details>
```

> 注意：块级 HTML 元素（如 `div`、`table`、`p`）内部的 Markdown 语法不会被处理。

## 转义字符

在特殊符号前加 `\` 可输出原义字符：

```markdown
\* 不是列表
\# 不是标题
\[不是链接]
```

支持转义的符号：

```
\ ` * _ { } [ ] ( ) # + - . !
```

## 复选框/任务列表

GFM（GitHub Flavored Markdown）扩展：

```markdown
## 待办事项
- [x] 已完成
- [ ] 未完成
- [ ] 另一个
```

## 注意事项

1. **英文输入**：所有 Markdown 语法符号请在英文输入法下输入
2. **嵌套使用**：所有语法都可嵌套，如列表套引用、引用内用标题等
3. **兼容性**：不同平台对 GFM 扩展的支持程度不一，建议以标准语法为主
4. **空行**：注意控制合适的空行间距，提高可读性
5. **语义优先**：优先使用有语义的标签（如 `>` 引用），而不是纯格式标签
