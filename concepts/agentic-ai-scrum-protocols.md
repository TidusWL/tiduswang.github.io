# Agentic AI 协议栈与人机混合协作

## 概述
当 AI Agent 从"单兵作战"走向"团队协作"时，**协议层**成为决定系统能否规模化运行的关键。2025-2026 年间，三类协议逐渐成为事实标准：MCP（Agent 调工具）、A2A（Agent 之间通信）、HITL（人介入）。它们与 [[ai-augmented-scrum]] 一起，构成现代 AI 团队工程的"水电煤"。

## 协议栈全景

```
┌─────────────────────────────────────────────┐
│  人机协作层 (Human-in-the-Loop / HITL)       │  ← 决策、审计、问责
├─────────────────────────────────────────────┤
│  智能体间通信 (A2A / ACP / AG-UI)            │  ← 多 Agent 任务交接
├─────────────────────────────────────────────┤
│  工具调用协议 (MCP)                          │  ← Agent → 外部系统
├─────────────────────────────────────────────┤
│  LLM 模型层 (Claude / GPT / Gemini)         │  ← 推理与生成
└─────────────────────────────────────────────┘
```

## 三大核心协议

### 1. MCP（Model Context Protocol）
- **作用**：AI Agent 调用外部工具、数据库、API 的标准化协议
- **类比**：相当于 AI Agent 时代的 "USB-C"——一套接口连所有外设
- **设计哲学**：把工具调用从"各 Agent 各自实现"变成"工具一次开发、所有 Agent 即插即用"
- 详见 [[mcp-protocol]]

### 2. A2A（Agent-to-Agent Protocol）
- **作用**：不同 Agent 之间传递任务、共享状态、协调冲突
- **关键能力**：
  - 动态任务分配（一个 Agent 失败，自动转给另一个）
  - 共享上下文（避免重复解析同一份资料）
  - 标准化错误处理
- **与 MCP 的区别**：MCP 是"Agent 调工具"，A2A 是"Agent 调 Agent"
- 同类协议：ACP（Agent Communication Protocol）、AG-UI、A2UI

### 3. HITL（Human-in-the-Loop）
- **作用**：在 Agent 自主执行链路中嵌入人类干预点
- **典型模式**：
  - **审批阈值**：金额 > X、影响 > Y 必须人审
  - **质量门禁**：AI 输出低于置信度阈值时回退人类
  - **可回滚/可审计**：所有 Agent 决策留痕，可回退到任意步骤
- **设计逻辑转向**：从"AI 能否执行"转向"**执行时谁负责**"——问责制成为 Agent 产品核心特性

## 部署架构演进

### 从单体到分布式
- **过去**：单一 AI 应用 + 单一模型
- **现在**：多 Agent 协作团队（主 Agent 拆解 + 子 Agent 执行）
- **未来（2026）**：Agent 控制平面 + 多 Agent 仪表盘，用户从单一入口管理所有 Agent 任务

### 经典多 Agent 架构
```
                 ┌──────────┐
                 │ Orchestrator │  ← 拆解目标、调度子 Agent
                 └─────┬────┘
        ┌──────────┬─────┴────┬──────────┐
        ▼          ▼          ▼          ▼
   ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐
   │数据Agent│ │内容Agent│ │分析Agent│ │报告Agent│  ← 各司其职
   └────────┘ └────────┘ └────────┘ └────────┘
```

麦肯锡预测 2026 年这类协作式智能体工作流将成为主流架构，核心衡量指标：
- 团队效率（端到端任务完成时间）
- 任务交接成功率（**无返工比例**）

## 人机混合团队的新角色

### Agent 团队负责人
- 2026 年新兴岗位，负责 Agent 资源调配、任务分配、质量管控
- IBM/Forrester 预测：每个员工都将转变为"智能体管理者"

### Hybrid Agency（混合代理）
- **定义**：人 + AI Agent + 机器人三方协作的工作模式
- 人类职责：长期规划、监管、价值判断
- AI 职责：执行、优化、反馈循环
- **不是 AI 替代人**，而是人转向更高层次的监督、设计与联合决策

## 与 Scrum 框架的连接

[[ai-augmented-scrum]] 的协议支撑：
- **Sprint Backlog 任务分配** → 通过 A2A 协议路由到合适的 Agent
- **Agent 调用工具**（git、CI、监控） → 通过 MCP 协议
- **Retrospective 中的人工审核** → 通过 HITL 协议留痕
- **Multi-Agent 协作的 Sprint** → 需要 Orchestrator + 子 Agent 模式

## 安全与治理

2026 年 Agent 部署的"必备能力栈"：
- 🔐 **身份验证**：Agent 也有自己的"身份证"
- 📊 **行为监控**：检测 Agent 异常行为（越权、误调用、token 失控）
- 🛑 **Kill-Switch**：紧急情况下立即关停单个 Agent 或整个集群
- 📜 **审计日志**：所有决策可回溯到具体的 Prompt、Context、Tool Call

## 开放问题
- 协议碎片化风险：MCP/A2A/ACP/AG-UI 谁会最终统一？
- **错误累积效应**：5 步任务每步 90% 准确率 → 整体 59%，长程任务如何稳定？
- Agent 自主性的边界：哪些决策可以完全自主，哪些必须保留人类"最终否决权"？
